# Expected findings

- Resolve `adaptive` from `PRODUCT.md` and keep product context separate from
  `DESIGN.md` visual authority.
- Read both iOS and Android quality guidance plus adaptive quality and
  interaction physics.
- Fail native trust on custom navigation/back handling, fixed phone canvas,
  undersized targets, disabled text scaling, raw colors, shared off-platform
  controls/icons, and phone-only tablet navigation.
- Recommend iOS navigation stack/tab/sidebar patterns and Android predictive
  Back/navigation bar-rail-drawer behavior rather than identical pixels.
- Require VoiceOver/TalkBack, Dynamic Type/font scaling, appearance,
  orientation, split-screen/multi-window, Reduced Motion/Remove animations,
  and representative runtime evidence.
- State that prompt-only evidence cannot prove compile success, runtime
  gestures, accessibility traversal, performance, or hardware feel.
