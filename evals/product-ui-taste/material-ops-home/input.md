# Product UI taste case: material ops home

## Purpose

Use this case to calibrate product UI taste scoring for a clean but generic
operations dashboard. It is based on a user-provided screenshot reviewed on
2026-06-30; the binary screenshot is intentionally not committed.

## Surface

- Page type: dashboard / operations home.
- Product: material asset workflow for upload, preview, AI analysis, scripts,
  platform video IDs, ad material IDs, authorization coverage, and callback
  matching.
- Audience: internal operators and admins.
- Primary job: identify what needs attention today and start the highest-value
  action quickly.
- Desired feel: calm, precise, data-dense, trustworthy, operational.

## Screenshot summary

- Left sidebar with product identity, nav groups, active Home item, and storage
  status.
- Top command bar with large search, upload CTA, management chip, and admin
  identity.
- Large hero: "Today's material items to process" with upload and library
  buttons.
- Twelve metric tiles in a 4-column grid:
  - total materials 2,061;
  - ready 2,043;
  - pending generation 0;
  - raw volume 26 GB;
  - pending source 18;
  - pending preview 0;
  - pending script 2027;
  - pending video ID 2024;
  - pending material ID 134;
  - authorization pending 2061;
  - AI failed tasks 1;
  - script failed tasks 1.
- Lower panels contain a workflow checklist and a "what to do first" action
  panel.

## Evidence level

L0 static screenshot only.

## Expected judgment

The UI is usable, aligned, and clean. It should not be scored as rough. It also
should not be scored above the mid-80s because the main task priority is flat,
the metric grid creates card soup, risk states are not visually differentiated,
and several labels expose internal system language.
